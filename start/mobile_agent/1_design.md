- 屏幕操作执行引擎
- 可能包含语音识别/TTS模块

## 针对老年人群体的技术升级方案

### 1. **更方便的唤醒机制**

**现状问题：**
- 可能需要打开APP或特定手势
- 老年人容易忘记操作步骤

**升级方案：**
- **多模态唤醒**
  - 持续后台监听唤醒词（如"小助手"、"帮帮我"）
  - 物理按键唤醒：音量键连按3次
  - 桌面悬浮球一键唤醒
  - 震动反馈确认唤醒成功
  
- **技术实现：**
  - 使用轻量级唤醒词检测模型（Porcupine/Snowboy）
  - Android Accessibility Service实现全局监听
  - 优化电池消耗，采用分级唤醒策略

### 2. **语音输入指令优化**

**现状问题：**
- 标准ASR对老年人口音、语速识别率低
- 方言支持不足

**升级方案：**
- **语音识别增强**
  - 集成多个ASR引擎（讯飞、百度、阿里）做融合识别
  - 针对老年人语音特征fine-tune：
    - 语速较慢的适配
    - 口齿不清的容错
    - 常见方言词汇库
  
- **交互优化：**
  - VAD（语音活动检测）延长等待时间
  - 支持多次补充说明（"再说一遍"、"我是说..."）
  - 语音播报识别结果，允许纠正

### 3. **上下文理解强化**

**现状问题：**
- 老年人指令往往不完整、跳跃性强
- 缺乏连贯的多轮对话能力

**升级方案：**
- **对话管理系统升级**
  - 扩展对话历史窗口（保留最近10轮对话）
  - 构建老年人常用场景的对话模板库：
    - "给儿子打电话" → 需要识别"儿子"是谁
    - "昨天那个..." → 需要关联昨天的操作
  
- **意图补全机制：**
  - Slot Filling策略：主动询问缺失信息
    - "您是要给谁打电话？"
    - "您要设置几点的提醒？"
  - 基于用户偏好的智能推断
  
- **技术实现：**
  - 在GLM prompt中注入：
    - 历史对话摘要
    - 用户偏好画像
    - 当前时间/位置上下文
  - Few-shot示例包含老年人典型对话模式

### 4. **反馈机制优化**

**现状问题：**
- 反馈不及时或过于技术化
- 老年人难以理解执行状态

**升级方案：**
- **多模态即时反馈**
  - **语音反馈**：简洁口语化
    - ✅ "好的，正在帮您打开微信"
    - ✅ "已经帮您发送了"
    - ❌ 避免："执行成功，返回码200"
  
  - **视觉反馈**：
    - 大字体、高对比度的状态显示
    - 执行步骤的动画提示
    - 成功/失败用绿色✓/红色✗图标
  
  - **触觉反馈**：
    - 操作成功：短震动
    - 需要注意：长震动
  
- **分步骤反馈：**
  - 复杂任务拆解："第一步...第二步...完成"
  - 进度条显示（如发送文件）

### 5. **用户偏好记录系统**

**升级方案：**
- **偏好数据库设计**
  ```
  - 联系人偏好：常联系的人、称呼映射
  - 时间偏好：常用的提醒时间点
  - APP使用习惯：高频应用、操作序列
  - 语言习惯：常用词汇、方言词
  - 健康数据：用药时间、作息规律
  ```

- **学习机制：**
  - 隐式学习：记录每次成功的指令-操作映射
  - 显式确认："以后'儿子'都是指张三，对吗？"
  - 定期总结：每周询问是否调整偏好

- **技术实现：**
  - 本地SQLite存储敏感偏好
  - 构建用户专属prompt模板
  - 使用Embedding索引相似历史场景

### 6. **健康生活提醒系统**

**功能扩展：**

- **用药提醒**
  - 多药品管理（名称、剂量、时间）
  - 提前5分钟语音+震动提醒
  - 吃药确认机制（语音确认/按钮确认）
  - 漏服提醒：30分钟后再次提醒
  - 药品库存管理：提醒购药

- **生活规律提醒**
  - **散步提醒**
    - 根据天气调整（接入天气API）
    - "天气不错，该出去走走了"
    - 计步器集成，达标鼓励
  
  - **吃饭提醒**
    - 三餐固定时间提醒
    - 学习用户用餐习惯
  
  - **睡眠提醒**
    - 就寝提醒："该准备休息了"
    - 勿扰模式自动开启
    - 起床提醒温和唤醒

- **技术实现：**
  - Android AlarmManager精准定时
  - WorkManager保证后台可靠执行
  - 提醒优先级管理（药物>睡眠>散步）
  - 智能免打扰（检测通话中、使用中）

## 需要注意的技术挑战

1. **电池续航**：后台监听需优化功耗
2. **隐私保护**：健康数据本地加密存储
3. **兼容性**：不同Android版本的Accessibility权限
4. **网络依赖**：离线场景下的降级方案
5. **误唤醒**：电视声音等干扰源的过滤
